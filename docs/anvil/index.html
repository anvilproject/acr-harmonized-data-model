version: None

Classes

  • Thing - Highest Level Class
    • AccessControlledRecord - Element for which specific access control restrictions apply.
      • SourceData - Submitted data about a particular Subject.
      • Subject - This entity is the subject about which data or references are recorded. | This includes the idea of a human participant in a study, a cell line, an animal model, | or any other similar entity.
        • Participant - Basic participant demographics
      • SubjectAssertion - Assertion about a particular Subject. May include Conditions, Measurements, etc.
    • AccessPolicy - Describes the access required for a given element of data.
    • Aliquot - Represent exact tubes or total amounts available of a Sample.
    • BiospecimenCollection - Biospecimen Collection
    • Family - Family
    • FamilyMember - Family
    • Measurement - Measurements
    • Procedure - Procedures
    • Sample - Biospecimen data
    • Study - Study Meta Data

Slots

  • access_policy_code - Access Policy Code
  • age_at_assertion - The age in decimal years of the Subject when the assertion was made.
  • age_at_collection - The age at which this biospecimen was collected.
  • age_at_event - The age in decimal years of the Subject at the time point which the assertion describes, | eg, age of onset or when a measurement was performed.
  • age_at_last_vital_status - Age at Last Vital Status
  • age_at_observation - Age of participant when measurement was taken/recorded
  • age_at_resolution - The age in decimal years of the Subject when the asserted state was resolved.
  • assertion_type - The semantic type of the resource, eg, Condition.
  • availablity_status - Can this Sample be requested for further analysis?
  • biospecimen_collection_id_fk - ID associated with the Biospecimen Collection event.
  • code - The structured term defining the meaning of the assertion.
  • consanguinity - Is there known or suspected consanguinity in this study family?
  • data_access_type - Data Access Type
  • date_of_birth - Date at which the individual was born. May be impacted by privacy rules described in date_of_birth_type.
  • date_of_birth_type - Privacy rule modification applied to date_of_birth.
  • description - Description
  • disease_limitation - Disease Use Limitations
  • display - The friendly display string of the coded term
  • ethnicity - Classification categories of human, taken from NIH, based on the social group a person belongs to, and either identifies with or is identified with by others, as a result of a complex of cultural, biological, geographical and other factors such as linguistic, dietary and religion traditions; ancestry, background, allegiance, or association; and physical characteristics traditionally associated with race.
  • ethnicity_source_value - Ethnicity value as reported in the original source
  • external_id - Other identifiers for this entity, eg, from the submitting study or in systems link dbGaP
  • external_study_id - External Study ID
  • family_description - Free title describing the study family, such as potential inheritance or details about consanguinity
  • family_external_id - Other study specific study identifiers
  • family_id_fk - Family Identifier
  • family_member_id - Family Member ID indicating presence in the family
  • family_study_focus - What is this study family investigating? EG, a specific condition. The code should be prefixed with a recognizable curie.
  • family_type - Describes the 'type' of study family, eg, trio.
  • funding_source - Funding Source
  • has_access_policy - Which access policy applies to this element?
  • has_assertion - Which assertions are made about this Subject?
  • has_sample - Which samples were collected or processed for this Subject?
  • id - ID associated with a class
  • in_study - With which study is this class associated?
  • laterality - Laterality information for the site
  • measurement_code - Measurement code should be from a recognized ontology preceded by a recognized curie. Multiple entries should reflect alternate codes for the same measurement.
  • measurement_source_value - Original measurement text
  • measurement_unit - UCUM Unit associated with the field
  • measurement_value_code - Code indicating measurement value such as positive, negative, etc.
  • method - The approach used to collect the biospecimen.
  • organism_type - Organism Type Label
  • other_family_member_id - Other member of the same family to which there is a relationship being described
  • parent_sample_id - Parent Sample ID
  • parent_study_id - Parent Study ID
  • phenotypic_sex - Sex of the Participant
  • phenotypic_sex_source_value - Original source value for phenotypic sex value
  • principal_investigator - Principal Investigator
  • procedure_code - Procedure code should be from a recognized ontology preceded by a recognized curie. Multiple entries should reflect alternate codes for the same procedure.
  • procedure_source_value - Original procedure text
  • processing - Curied code associated processing that was applied to the Parent Sample or from the Biospecimen Collection that yielded this distinct sample
  • quantity - The total quantity of the specimen
  • race - Reported race defined by NIH Racial and Ethnic Categories and Definitions (NOT-OD-15-089)
  • race_source_value - Race value as observed from the original source text
  • relationship_code - Relationship of the "Family Member" to the "Other Family Member" (i.e. mother, father, etc). This is required if other_family_member_id is present. Code must be from the HL7 FamilyMember ValueSet
  • sample_availability_status - Can this Sample be requested for further analysis
  • sample_concentration - What is the concentration of the analyte in the Aliquot?
  • sample_concentration_unit - Units associated with the concentration of the analyte in the Aliquot? UCUM coding preferred (with curie, UCUM)
  • sample_id_fk - Sample ID the aliquot is associated with
  • sample_type - Curied code associated the type of material of which this Sample is comprised
  • sample_volume - What is the volume of the Aliquot?
  • sample_volume_unit - Units associated with the volume? UCUM coding preferred (with curie, UCUM)
  • site - The location of the specimen collection.
  • source_data - The sources from which this assertion was derived
  • spatial_qualifier - Any spatial/location qualifiers
  • storage_method - Curied code indicating how is the Sample stored, eg, Frozen or with additives
  • study_title - Study Title
  • subject_type - Type of entity this record represents
  • value_code - The structured term defining the value of the assertion.
  • value_display - The friendly display string of the coded term for the value of the assertion.
  • value_number - The numeric value of the assertion.
  • value_units - The structured term defining the units of the value.
  • value_units_display - The friendly display string of units of the value.
  • vital_status - Vital Status
  • website - Website
